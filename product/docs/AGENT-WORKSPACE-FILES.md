# Agent Workspace Files

> **Why some files appear deleted from the repo source tree**

CoderClaw maintains two distinct categories of Markdown files that share similar
names but live in very different places:

---

## 1. Source-controlled templates (`docs/`)

These are versioned in the Git repository and document the project itself.
They travel with every clone and are visible in GitHub.

| File                   | Purpose                                     |
| ---------------------- | ------------------------------------------- |
| `docs/ARCHITECTURE.md` | System design, component map, data-flow     |
| `docs/CONTRIBUTING.md` | How to contribute, PR checklist             |
| `docs/SECURITY.md`     | Threat model, reporting, hardening guidance |
| `docs/VISION.md`       | Long-term product direction                 |

---

## 2. Runtime agent workspace files (user's `~/.coderclaw/workspace/` or `CWD`)

These are **created and maintained at runtime** by the running CoderClaw agent,
not by the source repository. They live in the **user's local workspace
directory** (typically `~/.coderclaw/workspace/` or the project root the agent
was initialised for) and are intentionally **not** committed to the repo.

| Filename               | Generated by                       | Purpose                                               |
| ---------------------- | ---------------------------------- | ----------------------------------------------------- |
| `SOUL.md`              | `coderclaw init` / agent self-init | Agent identity, values, and persona                   |
| `USER.md`              | Agent interaction loop             | User preferences and communication style              |
| `AGENTS.md`            | `coderclaw init`                   | Working conventions for the agent in this workspace   |
| `MEMORY.md`            | Agent memory loop                  | Long-term curated learnings (personal, not shared)    |
| `HEARTBEAT.md`         | Agent heartbeat system             | Periodic task reminders and background checks         |
| `TOOLS.md`             | Agent tool inventory               | Discovered tools, device names, SSH details           |
| `BOOTSTRAP.md`         | First-run bootstrap                | One-time setup instructions (deleted after first use) |
| `CLAUDE.md`            | Deprecated                         | Former name of `AGENTS.md`; replaced in 2025-Q4       |
| `USER_*.md`            | Multi-user workspaces              | Per-user preference files                             |
| `memory/YYYY-MM-DD.md` | Daily knowledge loop               | Short-term daily activity notes                       |

### Why they are not in the repo

- They contain **personal and workspace-specific data** (user preferences,
  API keys hints, device names) that must not be committed publicly.
- They are **generated fresh** per user and per workspace — a single template
  in the repo would be overwritten on every `coderclaw init` anyway.
- The `.gitignore` at the project root explicitly excludes them.

### Where BOOTSTRAP.md, CLAUDE.md, and USER.md went

These three files were referenced in the PR review as "deleted":

- **`BOOTSTRAP.md`** — was a hand-crafted file that existed in an early
  prototype version of this repository as a local dev convenience. The content
  (first-run setup instructions) moved into the `coderclaw init` command output
  and the `docs/CONTRIBUTING.md` development setup section. It is no longer
  stored as a static file because the bootstrap flow is now agent-generated.

- **`CLAUDE.md`** — was the original name for the agent working-conventions
  file (named after the Claude AI model). It was renamed to `AGENTS.md` to be
  model-agnostic. The rename happened in the workspace template layer; the
  repo-tracked copy was removed because the file is agent-generated, not
  source-controlled. Existing workspaces with a `CLAUDE.md` continue to work —
  the agent reads both names for backwards compatibility.

- **`USER.md`** — is a runtime-generated file that lives in the user's workspace
  root, not in the source repository. Its previous presence in the repo was an
  accidental commit of a developer's local workspace state. It has been removed
  from version control and added to `.gitignore`.

| ------------------------------------ | ---------------------------------------------------- |
